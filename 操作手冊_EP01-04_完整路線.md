# EP01-04 操作手冊：從零到 Docker 完整路線

> 本手冊涵蓋 WSL → VS Code → Git Clone → Python 爬蟲 → uv 環境 → Module/Package → Docker 的完整操作。
> 所有步驟都可以用 CLI 指令完成。每一步都標注你在哪個環境、哪個目錄。

---

## 環境標記說明

```
🪟 = Windows PowerShell（系統管理員）
🐧 = WSL Ubuntu Terminal
📁 = 當前目錄
```

## 第一段：安裝 WSL + Ubuntu（EP01）

### 🪟 Windows PowerShell（系統管理員）

```powershell
# 📁 任意目錄

# 1. 安裝 WSL 框架（不自動裝 Ubuntu）
wsl --install --no-distribution

# 2. 重新開機（必須）
# 手動重開機
```

### 🪟 重開機後，Windows PowerShell

```powershell
# 📁 任意目錄

# 3. 從 Microsoft Store 安裝 Ubuntu 22.04
#    打開 Microsoft Store → 搜尋 "Ubuntu 22.04" → 點「取得」
#    或用指令：
wsl --install -d Ubuntu-22.04

# 4. 確認安裝成功
wsl -l -v
# 預期輸出：
#   NAME            STATE           VERSION
# * Ubuntu-22.04    Running         2

# 5. 確認是 WSL2（不是 WSL1）
# VERSION 欄位要是 2
# 如果是 1：
wsl --set-version Ubuntu-22.04 2

# 6. 設定 WSL2 為預設
wsl --set-default-version 2
```

### 🐧 第一次打開 Ubuntu 22.04

```bash
# 📁 自動在 /home/你的名字

# 7. 設定使用者名稱（英文小寫）
# Enter new UNIX username: dex
# 輸入密碼（畫面不會顯示，正常）

# 8. 確認 Ubuntu 版本
cat /etc/os-release
# 預期：NAME="Ubuntu"  VERSION_ID="22.04"

# 9. 更新系統
sudo apt update && sudo apt upgrade -y

# 10. 確認 Python
python3 --version
# 預期：Python 3.10.x

# 11. 安裝 pip
sudo apt install python3-pip -y
pip3 --version

# 12. 確認 Git
git --version
# 如果沒有：sudo apt install git -y
```

## 第五段：uv 虛擬環境（EP02）

### 🐧 VS Code Terminal（WSL Ubuntu）

```bash
# 📁 ~/projects/de-test

# 23. 安裝 uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 24. 重新載入 shell（讓 uv 指令生效）
source ~/.bashrc
# 或
exec $SHELL

# 25. 確認 uv 安裝成功
uv --version
# 預期：uv 0.x.x

# 26. 進入 ep02 資料夾（已有 uv 設定）
cd ~/projects/de-test/ep02

# 27. 安裝套件（用 uv）
uv sync
# 預期：安裝 requests, pandas, beautifulsoup4, tqdm + 依賴

# 28. 用 uv 跑爬蟲
uv run python ptt_crawler.py
# 預期：印出 Stock 版文章

# 29. 用 uv 跑 finmind
uv run python finmind.py
# 預期：印出 0050 股價

# 30. 查看 uv 產生的檔案
cat .python-version
# 預期：3.13（或你的 Python 版本）

cat pyproject.toml
# 預期：看到 name, version, dependencies

ls uv.lock
# 預期：檔案存在（套件版本鎖定）
```

## 第六段：Module / Package（EP02）

### 🐧 VS Code Terminal（WSL Ubuntu）

```bash
# 📁 ~/projects/de-test/ep02

# 31. 看 crawlers package 結構
ls crawlers/
# 預期：__init__.py  finmind.py  hahow_crawler.py  ptt_crawler.py

# 32. 看 __init__.py 內容
cat crawlers/__init__.py
# 預期：
# from .ptt_crawler import ptt_crawler
# from .finmind import crawler_finmind
# from .hahow_crawler import crawler_hahow_course

# 33. 測試 package import（不會自動跑爬蟲）
uv run python -c "from crawlers import ptt_crawler; print(type(ptt_crawler))"
# 預期：<class 'function'>
# 注意：沒有印出爬蟲結果 ← if __name__ 生效

# 34. 手動呼叫函式
uv run python -c "
from crawlers import ptt_crawler
result = ptt_crawler('Tech_Job')
print(f'爬到 {len(result)} 篇文章')
"
# 預期：爬到 N 篇文章

# 35. 對比：直接 import 原始的 ptt_crawler.py（會自動跑）
cd ~/projects/de-test/ep02
uv run python -c "import ptt_crawler"
# 預期：直接印出 Stock 版文章 ← 因為原始檔底部沒有 if __name__

# 36. 回到 ep02 確認理解
# crawlers/ 裡的版本加了 if __name__ → import 安全
# 根目錄的版本沒有 → import 會自動跑
```

## 第七段：Docker（EP03）

### 🪟 Windows：安裝 Docker Desktop

```
1. 到 https://www.docker.com/products/docker-desktop/ 下載
2. 安裝時勾選 "Use WSL 2 instead of Hyper-V"
3. 安裝完重開機
4. 打開 Docker Desktop → Settings → Resources → WSL Integration
   → 把 Ubuntu-22.04 打開 → Apply & Restart
```

### 🐧 WSL Ubuntu Terminal（VS Code）

```bash
# 📁 ~/projects/de-test/ep03-04

# 37. 確認 Docker 可用
docker --version
# 預期：Docker version 2x.x.x

docker compose version
# 預期：Docker Compose version v2.x.x

# 如果 docker 找不到：
# → 確認 Docker Desktop 有開
# → 確認 WSL Integration 有打開 Ubuntu-22.04

# 38. 跑 hello-world 測試
docker run hello-world
# 預期：看到 "Hello from Docker!"

# 39. 看 container 和 image
docker ps -a
# 預期：看到 hello-world container（已停止）

docker images
# 預期：看到 hello-world image
```

### 🐧 Build 自己的 Image

```bash
# 📁 ~/projects/de-test/ep03-04

# 40. 看 Dockerfile 內容
cat Dockerfile

# 41. Build image
docker build -t de-test-app .
# 等待 build（第一次比較久）
# 預期最後一行：Successfully tagged de-test-app:latest

# 42. 確認 image
docker images
# 預期：看到 de-test-app

# 43. 跑 container（互動模式）
docker run -it de-test-app
# 現在你在 container 裡面了（提示符號變了）

# 📁 /app（container 內）

# 44. 在 container 裡跑爬蟲
uv run python crawlers/ptt_crawler.py
# 預期：跟在本機跑一樣的結果

# 45. 離開 container
exit
# 回到 WSL Ubuntu
```

## 第八段：Docker Compose 多服務（EP04）

### 🐧 WSL Ubuntu Terminal（VS Code）

```bash
# 📁 ~/projects/de-test/ep03-04

# 46. 看 docker-compose.yml
cat docker-compose.yml

# 47. 啟動所有服務（背景執行）
docker compose up -d
# 預期：
# ✔ Container rabbitmq   Started
# ✔ Container flower     Started
# ✔ Container mysql      Started
# ✔ Container phpmyadmin Started

# 48. 確認服務狀態
docker compose ps
# 預期：4 個服務都是 running

# 49. 打開瀏覽器驗證（在 Windows 瀏覽器）
# RabbitMQ 管理介面：http://localhost:15672（帳號 worker / 密碼 worker）
# Flower 監控：http://localhost:5555
# phpMyAdmin：http://localhost:8000（帳號 root / 密碼 1234）

# 50. 看特定服務的 log
docker compose logs -f rabbitmq
# Ctrl+C 離開 log

# 51. 只重啟某個服務
docker compose restart mysql

# 52. 停止所有服務
docker compose down
# 預期：所有 container 停止並移除

# 53. 確認都停了
docker ps
# 預期：沒有任何 container 在跑
```

## FAQ

### WSL 相關

**Q：wsl --install 出現錯誤 0x80370102**
A：BIOS 虛擬化沒開。重開機進 BIOS → 找 Intel VT-x 或 AMD-V → 啟用 → 存檔重開。

**Q：wsl -l -v 顯示 VERSION 是 1 不是 2**
A：`wsl --set-version Ubuntu-22.04 2`。如果失敗，確認虛擬化和 Windows 功能（虛擬機器平台）有開。

**Q：Windows 11 Home 版找不到 Hyper-V**
A：不需要 Hyper-V。WSL2 只需要「虛擬機器平台」（Virtual Machine Platform），`wsl --install` 會自動啟用。

**Q：Ubuntu 打不開 / 卡住**
A：PowerShell 打 `wsl --shutdown`，等 5 秒再開 Ubuntu。還不行就 `wsl --unregister Ubuntu-22.04` 重裝（⚠️ 資料全刪）。

**Q：登入是 root 不是個人帳號**
A：`ubuntu2204 config --default-user 你的帳號`。如果沒建過帳號：`adduser 你的帳號` → `usermod -aG sudo 你的帳號`。

### VS Code 相關

**Q：code . 沒反應**
A：Windows 重裝 VS Code，安裝時勾 "Add to PATH"。

**Q：VS Code 左下角不是 WSL: Ubuntu**
A：第一次 `code .` 時會自動提示裝 WSL Extension。如果沒有，手動到 Extensions 搜尋 "WSL" 安裝。

**Q：VS Code Server for WSL closed unexpectedly**
A：
1. PowerShell：`wsl --shutdown` → 重開
2. WSL 裡：`rm -rf ~/.vscode-server` → 重跑 `code .`
3. `wsl --update` → 重開機

**Q：之前裝的套件和 Python 版本不見了**
A：因為你現在在 WSL Ubuntu 環境裡，跟 Windows 是分開的。要在 Ubuntu 裡重新裝。

### Git 相關

**Q：git clone 失敗**
A：確認網路正常。如果是 private repo 需要 Token 或 SSH key。

**Q：push 被拒（rejected）**
A：遠端有新 commit。先 `git pull origin main` 再 push。

### uv 相關

**Q：uv: command not found**
A：跑 `curl -LsSf https://astral.sh/uv/install.sh | sh`，然後 `source ~/.bashrc`。

**Q：uv sync 失敗**
A：確認 pyproject.toml 存在。確認網路正常。

**Q：import 找不到套件**
A：用 `uv run python xxx.py` 而不是 `python3 xxx.py`。uv run 會自動使用虛擬環境。

### Docker 相關

**Q：docker: command not found（在 WSL 裡）**
A：Docker Desktop → Settings → Resources → WSL Integration → 把 Ubuntu-22.04 打開 → Apply & Restart。

**Q：Docker Desktop 卡在 Starting**
A：
1. 系統匣右鍵 Docker → Restart
2. PowerShell：`wsl --shutdown` → 重開 Docker Desktop
3. 確認虛擬化有開（工作管理員 → 效能 → CPU）

**Q：Docker Desktop 吃太多記憶體**
A：建 `%USERPROFILE%\.wslconfig`：
```
[wsl2]
memory=3GB
processors=2
swap=1GB
```
然後 `wsl --shutdown`。

**Q：docker compose up 報 port already in use**
A：`docker ps` 找到佔用的 container → `docker stop <id>`。或改 docker-compose.yml 的 port。

**Q：permission denied**
A：`sudo usermod -aG docker $USER` → 重開 Terminal。

**Q：build 很慢**
A：第一次要下載基底 image（幾百 MB），之後有快取會快。確認網路穩定。
