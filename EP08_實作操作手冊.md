## 第一部分：啟動共用基礎設施

多 Worker 要能一起搶任務、把資料寫進同一個 DB，前提是大家都在**同一張網路**上。

### Step 1：建立共用網路 my_network

`rabbitmq-network.yml`、`mysql.yml`、Worker、Producer **全部都掛在 `my_network` 這張外部網路**，
彼此才能用「服務名稱」互相找到（例如 Worker 用 `rabbitmq` 找到佇列、用 `mysql` 找到資料庫）。

```bash
docker network create my_network
```

✅ **預期**：印出一串網路 ID（如果出現 `already exists` 代表先前建過，忽略即可）。

```bash
# 確認網路存在
docker network ls | grep my_network
```

### Step 2：啟動 RabbitMQ + Flower

```bash
cd ~/stock-crawler
docker compose -f rabbitmq-network.yml up -d
```

✅ **預期**：`rabbitmq` 和 `flower` 兩個容器啟動。

```bash
docker compose -f rabbitmq-network.yml ps
```

### Step 3：啟動 MySQL（讓爬蟲任務有地方寫資料）

本集主角是「多 Worker / 多 Queue」，但我們發的是真實爬蟲任務 `crawler_finmind`，
它爬完會把資料寫進 MySQL，所以要先把 MySQL 開起來（DB 細節留到 EP09）。

```bash
docker compose -f mysql.yml up -d
docker compose -f mysql.yml logs mysql | tail -10
```

✅ **預期**：看到 `ready for connections` 才代表資料庫可以連了。

> ⚠️ 常見坑：MySQL 第一次啟動要初始化十幾秒，剛 `up` 馬上連會 `Connection refused`，等 `ready for connections` 再繼續。

### Step 4：驗證三個 Web 介面

```bash
curl -s -o /dev/null -w "RabbitMQ  %{http_code}\n" http://localhost:15672
curl -s -o /dev/null -w "Flower    %{http_code}\n" http://localhost:5555
curl -s -o /dev/null -w "phpMyAdmin %{http_code}\n" http://localhost:8080
```

✅ **預期**：三個都回 `200`。

## 第二部分：啟動多 Worker Compose

### Step 1：看懂 docker-compose-worker-network.yml

```bash
cat docker-compose-worker-network.yml
```

重點是它定義了**兩個 Worker 服務**，各自只消費一條 Queue：

```yaml
services:
  crawler_twse:
    image: linsamtw/tibame_crawler:0.0.2
    hostname: "twse"
    command: uv run python -m celery -A crawler.worker worker --loglevel=info --hostname=worker@%h -Q twse
    environment:
      - RABBITMQ_HOST=rabbitmq    # 用服務名連 RabbitMQ
      - MYSQL_HOST=mysql          # 用服務名連 MySQL
    networks:
      - my_network

  crawler_tpex:
    image: linsamtw/tibame_crawler:0.0.2
    hostname: "tpex"
    command: uv run python -m celery -A crawler.worker worker --loglevel=info --hostname=worker@%h -Q tpex
    environment:
      - RABBITMQ_HOST=rabbitmq
      - MYSQL_HOST=mysql
    networks:
      - my_network
```

| 參數 | 作用 |
|------|------|
| `-Q twse` / `-Q tpex` | 這個 Worker **只消費**指定的 Queue，不碰其他 Queue |
| `--hostname=worker@%h` | `%h` 自動帶入容器 hostname，讓每個 Worker 在 Flower 有獨立名字 |
| `RABBITMQ_HOST=rabbitmq` | 覆蓋 config.py 預設的 `127.0.0.1`，用 Docker 服務名連線 |
| `networks: my_network` | 接上共用網路，才連得到 rabbitmq / mysql |

### Step 2：啟動兩個 Worker

```bash
docker compose -f docker-compose-worker-network.yml up -d
```

✅ **預期**：`crawler_twse`、`crawler_tpex` 兩個容器啟動。

```bash
docker compose -f docker-compose-worker-network.yml ps
```

看到兩個服務都是 `Up`。

### Step 3：看 Worker log 確認 ready 並認得 task

```bash
docker compose -f docker-compose-worker-network.yml logs | tail -40
```

✅ **預期**：
- 印出環境變數 `RABBITMQ_HOST: rabbitmq`（不是 127.0.0.1）
- `[tasks]` 區塊列出 `crawler.tasks_crawler_finmind.crawler_finmind`
- 每個 Worker 最後一行 `celery@twse ready.` / `celery@tpex ready.`

> ⚠️ 啟動初期可能短暫出現連 RabbitMQ 的 `Connection refused`，那是啟動順序造成的，看到 `ready` 就沒事。

## 第三部分：發任務到多 Queue

### Step 1：看懂 producer_multi_queue.py

```bash
cat crawler/producer_multi_queue.py
```

核心邏輯：用 `.s()` 把 task 和參數綁成 signature，再用 `apply_async(queue=)` 指定送哪條 Queue。

```python
from crawler.tasks_crawler_finmind import crawler_finmind

# .s() = signature，把 task + 參數綁成一個可派送的物件
task_2330 = crawler_finmind.s(stock_id="2330")
task_2330.apply_async(queue="twse")        # 上市股票 → twse queue
print("send task_2330 task")

task_00679b = crawler_finmind.s(stock_id="00679B")   # 美債
task_00679b.apply_async(queue="tpex")      # → tpex queue
print("send task_00679b task")
```

> 📌 關鍵對應：Producer 送去 `queue="twse"` 的任務，只有啟動時加了 `-Q twse` 的 Worker 才會消費。
> queue 名稱對不上 → 任務沒人處理，會一直卡在 RabbitMQ。

### Step 2：用 Producer Compose 發任務（推薦）

`docker-compose-producer-network.yml` 會在同一張網路裡跑 `producer_multi_queue.py`：

```bash
docker compose -f docker-compose-producer-network.yml up
```

✅ **預期**：印出 `send task_2330 task` / `send task_00679b task` 後容器結束。

> 一次性服務跑完 `Exited(0)` 是正常的，Producer 的工作就是「發完任務就走」。

### Step 3（替代方案）：直接在本機發任務

如果想在本機跑（不透過容器），記得本機要連得到 RabbitMQ（rabbitmq-network.yml 有開 5672）：

```bash
uv run python crawler/producer_multi_queue.py
```

✅ **預期**：一樣印出兩行 `send ... task`。

## 第四部分：Flower 觀察多 Worker 分工

### Step 1：看 Worker 各自收到任務

```bash
docker compose -f docker-compose-worker-network.yml logs | tail -40
```

✅ **預期**：
- `crawler_twse` 的 log 出現處理 `2330` 的任務並 `succeeded`
- `crawler_tpex` 的 log 出現處理 `00679B` 的任務並 `succeeded`
- 兩檔股票被**分流到不同 Worker**（這就是多 Queue 的效果）

### Step 2：Flower Web 介面確認

瀏覽器打開 `http://localhost:5555`：

| 頁籤 | 看什麼 |
|------|--------|
| **Workers** | 看到 `celery@twse`、`celery@tpex` 兩個 Worker 都 Online |
| **Tasks** | 看到 2 筆 `crawler_finmind`，狀態都是 **SUCCESS** |

點進每個 Worker 可以看到它各自 **Processed** 了幾個任務，確認任務真的有分流。

✅ **預期**：兩個 Worker 各處理 1 筆，狀態 SUCCESS（不是 FAILURE / PENDING）。

> 💡 如果某個任務一直 PENDING：通常是 Producer 送的 queue 名稱和 Worker 的 `-Q` 對不上，
> 回頭檢查 `apply_async(queue=...)` 和 compose 裡的 `-Q`。

## 第五部分：docker compose scale 動態擴充

一個 twse Worker 還是可能不夠。`--scale` 可以一行指令長出多個一模一樣的 Worker。

### Step 1：把 crawler_twse 擴充成 3 個

```bash
docker compose -f docker-compose-worker-network.yml up -d --scale crawler_twse=3
```

✅ **預期**：Docker 啟動 `crawler_twse-1`、`crawler_twse-2`、`crawler_twse-3` 三個容器。

```bash
docker compose -f docker-compose-worker-network.yml ps
```

看到 3 個 twse Worker + 1 個 tpex Worker。

### Step 2：為什麼 worker 可以 scale？

| 前提 | 說明 |
|------|------|
| 不對外開 port | Worker 不像 rabbitmq/mysql 要對外開 port，多開幾個不會 port 打架 |
| 無狀態 stateless | 任務從 RabbitMQ 拿、結果寫 MySQL，Worker 自己不存狀態，所以可複製 |
| 共用 broker | 三個 twse Worker 連同一個 RabbitMQ，一起搶 twse queue 的任務 |

> ⚠️ `rabbitmq` 和 `mysql` **不能** scale（有狀態、要對外開固定 port），但 worker 可以。

### Step 3：發多一點任務看分工

多發幾檔上市股票，觀察 3 個 twse Worker 一起消化：

```bash
uv run python -c "
from crawler.tasks_crawler_finmind import crawler_finmind
for stock_id in ['2330', '2317', '2454', '2412', '1301', '1303']:
    crawler_finmind.s(stock_id=stock_id).apply_async(queue='twse')
    print('send', stock_id)
"
```

```bash
docker compose -f docker-compose-worker-network.yml logs crawler_twse | tail -40
```

✅ **預期**：6 檔股票被 **3 個 twse Worker 分著做**（每個約做 2 檔），總時間比 1 個 Worker 快。
在 Flower 的 Workers 頁可看到三個 twse Worker 的 Processed 數字都增加。

### Step 4：縮回去 / 關閉服務

```bash
# 縮回 1 個 twse worker
docker compose -f docker-compose-worker-network.yml up -d --scale crawler_twse=1

# 全部關閉
docker compose -f docker-compose-worker-network.yml down
docker compose -f rabbitmq-network.yml down
docker compose -f mysql.yml down          # data volume 會保留
```

> 💡 `down` 不會刪 volume，下次 `up` 資料還在；要連資料一起清才用 `down -v`（謹慎使用）。
