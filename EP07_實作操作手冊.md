## 第一部分：啟動完整 infra（RabbitMQ + MySQL）

EP06 只開了 RabbitMQ。EP07 要同時開 RabbitMQ（訊息佇列）和 MySQL（資料庫），
所以用「帶網路」的版本，讓兩組服務掛在同一個網路上、能互相找到對方。

### Step 1：建立共用 Docker 網路

`rabbitmq-network.yml` 和 `mysql.yml` 都用 `external: true`，代表網路要**先手動建好**。

```bash
docker network create my_network
```

✅ **預期**：印出一段網路 ID（例如 `a1b2c3d4...`）

```bash
# 確認網路已存在
docker network ls | grep my_network
```

> 💡 如果出現 `network with name my_network already exists`，代表之前建過了，直接略過。

### Step 2：啟動 RabbitMQ + Flower

```bash
cd ~/stock-crawler
docker compose -f rabbitmq-network.yml up -d
```

✅ **預期**：`rabbitmq` 和 `flower` 兩個服務都 Started

```bash
docker compose -f rabbitmq-network.yml ps
```

✅ **預期**：兩個都是 `Up`。

> `rabbitmq-network.yml` 啟動兩個服務：
> - **rabbitmq**：佇列本體，AMQP 埠 `5672`、管理介面 `15672`，帳密 `worker / worker`
> - **flower**：Celery 監控 Web UI，埠 `5555`

### Step 3：啟動 MySQL + phpMyAdmin

```bash
docker compose -f mysql.yml up -d
```

✅ **預期**：`mysql` 和 `phpmyadmin` 兩個服務都 Started

```bash
docker compose -f mysql.yml ps
```

> `mysql.yml` 啟動兩個服務：
> - **mysql**：資料庫本體，埠 `3306`，預設資料庫 `mydb`
> - **phpmyadmin**：MySQL 的網頁操作介面，埠 `8080`（容器內 80）

### Step 4：確認三個 Web 介面都通

```bash
curl -s -o /dev/null -w "RabbitMQ:   %{http_code}\n" http://localhost:15672   # 預期 200
curl -s -o /dev/null -w "Flower:     %{http_code}\n" http://localhost:5555    # 預期 200
curl -s -o /dev/null -w "phpMyAdmin: %{http_code}\n" http://localhost:8080    # 預期 200
```

| 服務 | 網址 | 帳密 |
|------|------|------|
| RabbitMQ 管理介面 | http://localhost:15672 | worker / worker |
| Flower 監控 | http://localhost:5555 | （無） |
| phpMyAdmin | http://localhost:8080 | root / 1234 |

> ⚠️ MySQL 第一次啟動需要幾秒鐘初始化。若 phpMyAdmin 一開始連不上，等 10 秒再 curl 一次。
> phpMyAdmin 帳密就是 `mysql.yml` 裡 `MYSQL_ROOT_PASSWORD` 設的值（root）；也可用 `user / 1234` 登入。

## 第二部分：RabbitMQ 管理介面（:15672）

這個介面看的是「訊息」——訊息有沒有在排隊、哪個 Queue 積壓了、誰連上來了。

### Step 1：登入

瀏覽器開 http://localhost:15672

- 帳號 / 密碼：`worker` / `worker`

✅ **預期**：登入後進到 **Overview** 頁。

### Step 2：逛 Overview 頁

Overview 是系統健康總覽，先看這幾塊：

| 區塊 | 看什麼 |
|------|--------|
| Totals（Queued messages） | Ready（待處理）、Unacked（處理中）、Total 三個數字 |
| Message rates | publish（進來）、deliver（送出）每秒幾則 |
| Global counts | 目前有幾條 Connections、Channels、Queues |
| Nodes | RabbitMQ 伺服器的記憶體、磁碟、運作狀態 |

> 💡 現在還沒發任務，所以訊息數應該是 0。先記住這個「空的樣子」，第五部分發任務後再回來對照。

### Step 3：看 Queues 分頁

點上方 **Queues**（或 Queues and Streams）分頁。

✅ **預期**：現在可能只有 Celery 預設的 queue，或是空的。重點先認得這幾個欄位：

| 欄位 | 看什麼 |
|------|--------|
| Name | 佇列名稱（celery / twse / tpex …） |
| Ready | 排隊中、還沒被領走的訊息數 |
| Unacked | 已被 Worker 領走、還沒回報完成的訊息數 |
| Total | Ready + Unacked |

> 🐛 **排錯第一招**：之後若「發了任務卻沒結果」，來這頁看 Ready。
> Ready 一直累積不下降 = 任務送到了一個**沒有 Worker 在聽**的 Queue（queue 名稱對不上）。

### Step 4：看 Connections 分頁

點 **Connections** 分頁。

✅ **預期**：目前至少會看到 **flower** 的連線（它一直連著 RabbitMQ 在監控）。
等第五部分啟動 Worker 後，這裡會多出 Worker 的連線。

> 💡 Connections 是「連線」，點進去還能看到裡面的 Channels（通道）。
> 一般練習看到連線有出現、代表程式真的連上了，就夠了。

## 第三部分：Flower 監控（:5555）

RabbitMQ 介面看「訊息」，Flower 看「任務和 Worker」。兩個角度互補。

### Step 1：打開 Flower

瀏覽器開 http://localhost:5555（免帳密）

### Step 2：看 Workers 分頁

✅ **預期**：現在還沒啟動 Worker，所以 Workers 列表可能是空的。
這很正常——第五部分啟動 Worker 後，這裡就會出現一列。

先認得欄位：

| 欄位 | 看什麼 |
|------|--------|
| Worker name | Worker 名字（celery@主機名） |
| Status | Online / Offline |
| Active | 現在正在執行幾個任務 |
| Processed | 從啟動到現在累積處理了幾個任務 |

### Step 3：認識 Tasks 分頁

點 **Tasks** 分頁。現在是空的，但先記住任務會有這幾種狀態：

| 狀態 | 意義 |
|------|------|
| PENDING | 等待中，還沒被領走 |
| STARTED | Worker 已開始執行 |
| SUCCESS | 成功完成 |
| FAILURE | 執行出錯（紅色） |

> 💡 任務失敗時，點進那筆 FAILURE，可以看到 **Traceback**（Python 錯誤堆疊）和當初的 **Args / Kwargs**。
> 比翻一大堆終端機 log 快很多——這是 Flower 最實用的功能。

## 第四部分：phpMyAdmin（:8080）

前兩個介面看「任務怎麼跑」，phpMyAdmin 看「資料最後存到哪」。
EP07 只要先會登入、看得到資料庫就好；真正寫入資料是 EP08，深入 SQL 是 EP09。

### Step 1：登入

瀏覽器開 http://localhost:8080

| 登入欄位 | 值 |
|---------|----|
| 伺服器 | mysql（mysql.yml 內 PMA_HOST 已設好，通常不用填） |
| 使用者名稱 | root |
| 密碼 | 1234 |

✅ **預期**：登入後看到 phpMyAdmin 主畫面，左側欄列出資料庫。

### Step 2：瀏覽資料庫

1. 左側欄點開資料庫 **mydb**
2. 看裡面的資料表（Tables）

✅ **預期**：EP07 還沒寫入任何爬蟲資料，所以 `mydb` 裡可能**沒有資料表、或資料表是空的**。
這完全正常——EP08 把爬蟲結果寫進來後，這裡才會出現股價資料表。

> 💡 之後有資料表時：點資料表 → **Browse** 分頁看每一列；**Structure** 分頁看欄位定義。

## 第五部分：發任務，在三個介面觀察

infra 和介面都熟悉了，現在發一批任務，**在三個介面同時觀察訊息怎麼流動**。

### Step 1：啟動一個本機 Worker

EP07 不開容器 Worker，用最簡單的本機 Worker 即可。**開一個終端機**，掛著別關：

```bash
cd ~/stock-crawler
uv run python -m celery -A crawler.worker worker --loglevel=info -E
```

> `-E` = 送出任務事件，這樣 Flower 的 Tasks 分頁才看得到每一筆任務明細。

✅ **預期**：看到 Celery 啟動橫幅，最後 `celery@你的主機名 ready.`。

### Step 2：（介面一）回 RabbitMQ 看連線變化

回 http://localhost:15672 → **Connections** 分頁。

✅ **預期**：除了 flower，現在**多出一條 Worker 的連線**。代表 Worker 真的連上 RabbitMQ 了。

### Step 3：（介面二）回 Flower 看 Worker 上線

回 http://localhost:5555 → **Workers** 分頁。

✅ **預期**：看到剛剛啟動的 Worker，Status 是 **Online**。

### Step 4：發一批 print 版爬蟲任務

**另外開第二個終端機**（保留 Worker 終端機不動）：

```bash
cd ~/stock-crawler
uv run python crawler/producer_crawler_finmind_print.py
```

✅ **預期**：依序印出 5 個股票代碼，程式秒退（任務已丟進 RabbitMQ）：

```
2330
0050
2317
0056
00713
```

### Step 5：（介面一）RabbitMQ Overview / Queues 看訊息流動

回 http://localhost:15672：

- **Overview** 的 Message rates：publish / deliver 會跳動一下
- **Queues** 分頁：對應 queue 的 Ready 數字短暫上升後被 Worker 消化掉、回到 0

✅ **預期**：看得到「訊息進來 → 被送出」的瞬間變化。這就是 RabbitMQ 在做的事。

### Step 6：（介面二）Flower 看任務結果

回 http://localhost:5555 → **Tasks** 分頁。

✅ **預期**：看到 5 筆 `crawler_finmind_print` 任務，狀態由 `RECEIVED` → `STARTED` → **SUCCESS**。
點任一筆進去，可看到 Args / Kwargs（哪支股票）和 Runtime（花了幾秒）。

也回 Worker 終端機看一眼，會看到 5 支股票的 FinMind 股價 DataFrame 被印出來、最後 `succeeded`。

### Step 7：（介面三）phpMyAdmin 確認「還沒有資料」

回 http://localhost:8080 → `mydb`。

✅ **預期**：print 版**不寫 DB**，所以這裡依然沒有股價資料——符合預期。
這正好說明三個介面的分工：任務成功（Flower）≠ 資料落地（phpMyAdmin）。資料寫入是 EP08 的事。

> 🐛 **若 Flower 看不到任務**：確認 Worker 啟動時有加 `-E`；沒加的話 Tasks 分頁會是空的（但 Workers 分頁仍看得到 Worker）。

## 第六部分：Portainer（:9000）

Portainer 是 Docker 的圖形化管理介面，讓你在瀏覽器裡看到所有 container 的狀態。

### Step 1：安裝並啟動 Portainer

```bash
docker run -d \
  -p 9000:9000 \
  --name portainer \
  --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  portainer/portainer-ce:latest
```

✅ **預期**：回傳一串 container ID

### Step 2：首次登入設定

打開瀏覽器 → `http://localhost:9000`

1. 設定管理員帳號（例：admin）
2. 設定密碼（至少 12 碼，例：`Admin1234567!`）
3. 點 **Get Started**
4. 選擇 **local** 環境 → 進入 Dashboard

### Step 3：瀏覽 Container 列表

1. 左側選單 → **Containers**
2. 可以看到所有 container 的：
   - **Status**：Running / Stopped
   - **Image**：用的映像檔
   - **Ports**：端口映射
   - **Created**：建立時間

### Step 4：單一 Container 操作

點進任一 container（例如 rabbitmq），可以：

| 功能 | 說明 |
|------|------|
| **Logs** | 看 container 的 log（和 `docker logs` 一樣，但有 GUI） |
| **Console** | 進入 container 的 shell（和 `docker exec -it` 一樣） |
| **Inspect** | 看 container 的詳細設定（env、volumes、network） |
| **Restart** | 重啟 container |
| **Stop / Kill** | 停止 container |

### Step 5：Portainer vs docker 指令

| 操作 | docker 指令 | Portainer |
|------|------------|-----------|
| 看 container | `docker ps -a` | Containers 列表 |
| 看 log | `docker logs <name>` | 點 container → Logs |
| 進入 shell | `docker exec -it <name> bash` | 點 container → Console |
| 重啟 | `docker restart <name>` | 點 container → Restart |
| 看全貌 | 要下多個指令 | 一個 Dashboard 全看到 |

### 驗證

```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:9000
```

✅ **預期**：`200`

## 第七部分：排錯 SOP — 四個介面怎麼搭配

系統出問題時，按這個順序檢查：

| 順序 | 介面 | 看什麼 | 常見問題 |
|:----:|------|--------|---------|
| 1 | **Flower**（:5555） | Task 是 SUCCESS 還是 FAILURE | 如果 FAILURE → 點進去看 traceback |
| 2 | **RabbitMQ**（:15672） | Queue 有沒有堆積 | Messages 越來越多 = Worker 消化不了 |
| 3 | **Portainer**（:9000） | Container 有沒有 crash / restart | 如果一直 Restarting → 看 Logs |
| 4 | **phpMyAdmin**（:8080） | 資料有沒有進 DB | 如果沒資料 → Worker 可能沒寫入成功 |

```bash
# 也可以用 docker compose logs 看詳細 error
docker compose -f rabbitmq-network.yml logs --tail 50
```

## 收工：關閉服務

```bash
cd ~/stock-crawler

# 先回 Worker 終端機按 Ctrl + C 停掉本機 Worker

# 停掉兩組 infra
docker compose -f mysql.yml down
docker compose -f rabbitmq-network.yml down

# 網路可留著下次用，要清除才執行：
# docker network rm my_network
```

✅ **預期**：所有容器被移除。

> 💡 `mysql.yml` 用了 named volume（`mysql:`），所以 `down` 之後資料還在；
> 要連資料一起清除才加 `-v`：`docker compose -f mysql.yml down -v`。

## 常見問題（Troubleshooting）

| 症狀 | 原因 | 解法 |
|------|------|------|
| `up` 報找不到網路 my_network | 還沒建立外部網路 | 先 `docker network create my_network` |
| phpMyAdmin 連不上 / 登入失敗 | MySQL 還在初始化 / 密碼打錯 | 等 10 秒再試；密碼用 `1234` |
| RabbitMQ Queues 的 Ready 一直累積 | 任務送到沒有 Worker 在聽的 queue | 確認 Worker 有啟動、queue 名稱對得上 |
| Flower Workers 是空的 | Worker 沒啟動 / 沒連上 | 確認 Worker 終端機有 `ready.`、RabbitMQ Connections 有它 |
| Flower Tasks 是空的 | Worker 沒送事件 | 啟動 Worker 時加 `-E` |
| 埠被占用（5672 / 5555 / 3306 / 8080） | 之前的容器還開著 | 先把舊的 compose `down` 再重開 |

## 下集預告：EP08 多 Worker + 多 Queue

EP07 我們把三個介面看熟了。EP08 開始真的「擴展」：

4. 用正式版 producer 把爬蟲結果真正寫進 MySQL，回 phpMyAdmin 驗證資料落地
