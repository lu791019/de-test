## 第一部分：Docker 單一容器操作

> 📖 完整操作見 [Docker Compose 開發操作手冊](Docker_Compose開發操作手冊.md)

### Step 1：啟動 Docker

```bash
sudo service docker start
sudo service docker status
```

✅ **預期**：`Active: active (running)`

### Step 2：用 docker run 跑 Nginx

```bash
docker run -d --rm --name my-nginx -p 8080:80 nginx:stable-alpine
```

✅ **驗證**：

```bash
curl http://localhost:8080
```

看到 `Welcome to nginx!` = Docker 正常。

注意你打了多少參數：`-d`（背景）、`--rm`（用完刪）、`--name`（命名）、`-p 8080:80`（port 映射）、`nginx:stable-alpine`（image）。

```bash
# 停止
docker stop my-nginx
```

## 第二部分：Docker Compose 操作

### Step 1：Clone nginx-demo 專案

```bash
cd ~
git clone https://github.com/DataEngCamp/nginx-demo.git
cd nginx-demo
```

### Step 2：看 docker-compose.yaml

```bash
cat docker-compose.yaml
```

```yaml
services:
  nginx:
    image: nginx:stable-alpine
    container_name: my-nginx
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html
    restart: unless-stopped
```

和剛才 `docker run` 一樣的事，但寫在 YAML 裡，清楚好管理。

### Step 3：用 docker compose 啟動

```bash
docker compose up -d
```

✅ **驗證**：

```bash
docker compose ps
curl http://localhost:8080
```

看到 `It works! 🚀`（nginx-demo 的自訂首頁）。

### Step 4：停止

```bash
docker compose down
```

### docker run vs docker compose 比較

| | docker run | docker compose |
|---|---|---|
| **設定方式** | 全部寫在一行指令裡 | 寫在 YAML 檔案裡 |
| **可讀性** | 參數一長串，難讀 | 結構清楚，一目了然 |
| **多服務** | 每個服務各打一行 | 一個 `docker compose up` 全部起來 |
| **適合** | 快速測試單一容器 | 正式開發、多服務系統 |

> 接下來的課程都用 Docker Compose 管理多服務。

## 第四部分：認識專案結構

```
stock-crawler/
├── crawler/                          ← Python Package（所有程式碼在這）
│   ├── worker.py                     ← Celery app 定義
│   ├── config.py                     ← 環境變數管理
│   ├── tasks.py                      ← 簡單範例 task（print）
│   ├── tasks_crawler_finmind.py      ← 真實爬蟲 task（FinMind API）
│   ├── producer.py                   ← 最簡單的發任務（1 次）
│   ├── producer_crawler_finmind.py   ← 批次發爬蟲任務（for 迴圈）
│   ├── producer_crawler_finmind_print.py ← 批次發（print 版，不寫 DB）
│   ├── producer_multi_queue.py       ← 多 queue 分流
│   ├── scheduler.py                  ← 定時排程（APScheduler）
│   └── upload_data_to_mysql.py       ← 資料寫入 MySQL
├── rabbitmq.yml                      ← RabbitMQ + Flower（Broker）
├── mysql.yml                         ← MySQL + phpMyAdmin（Database）
├── docker-compose-worker.yml         ← Worker 服務
├── docker-compose-worker-network.yml ← 多 Worker（twse/tpex 分流）
├── docker-compose-producer-network.yml ← Producer 服務
├── Dockerfile                        ← 容器化 Worker
├── pyproject.toml + uv.lock          ← Python 依賴管理
└── README.md                         ← 學習順序 1-6 步
```

### 漸進式學習路徑

stock-crawler 的設計是「每個概念都有簡化版先跑通」：

| 階段 | 簡化版（先跑通） | 正式版（再深入） |
|------|----------------|----------------|
| Task | `tasks.py`（只 print） | `tasks_crawler_finmind.py`（真的爬 API） |
| Producer | `producer.py`（發 1 次） | `producer_crawler_finmind.py`（for 迴圈批次） |
| Producer（不寫 DB） | `producer_crawler_finmind_print.py` | `producer_crawler_finmind.py`（含 MySQL 寫入） |
| Queue | 預設單一 queue | `producer_multi_queue.py`（twse/tpex 分流） |
| 排程 | 手動執行 | `scheduler.py`（APScheduler 定時） |

## 第五部分：跑第一個 FinMind 爬蟲

### Step 1：安裝 Python 套件

```bash
cd ~/stock-crawler
uv sync
```

✅ **預期**：安裝 celery、requests、pandas、loguru 等套件（根據 pyproject.toml）

### Step 2：測試 FinMind API（不用啟動 Docker）

這一步直接用 Python 呼叫 FinMind API，確認 API 能通：

```bash
uv run python -c "
import requests
import pandas as pd

url = 'https://api.finmindtrade.com/api/v4/data'
parameter = {
    'dataset': 'TaiwanStockPrice',
    'data_id': '2330',
    'start_date': '2025-01-01',
    'end_date': '2025-06-28',
}
resp = requests.get(url, params=parameter)
data = resp.json()
if resp.status_code == 200:
    df = pd.DataFrame(data['data'])
    print(f'抓到 {len(df)} 筆資料')
    print(df.head())
else:
    print(data['msg'])
"
```

✅ **預期**：

```
抓到 XXX 筆資料
       date stock_id  Trading_Volume  ...  close  spread  Trading_turnover
0  2025-01-02     2330       34587892  ...  1040.0    15.0     35810803350
1  2025-01-03     2330       28374881  ...  1050.0    10.0     29720832960
...
```

看到台積電（2330）的股價 DataFrame = FinMind API 能通。

### Step 3：看懂回傳的欄位

| 欄位 | 說明 |
|------|------|
| `date` | 日期 |
| `stock_id` | 股票代碼 |
| `Trading_Volume` | 成交量（股） |
| `open` | 開盤價 |
| `max` | 最高價 |
| `min` | 最低價 |
| `close` | 收盤價 |
| `spread` | 漲跌 |
| `Trading_turnover` | 成交金額 |

## 第六部分：看懂程式碼

### config.py — 環境變數管理

```bash
cat crawler/config.py
```

重點：用 `os.environ.get(key, default)` 讀環境變數。開發時用預設值，Docker 裡用 environment 覆蓋。

| 變數 | 預設值 | 用途 |
|------|--------|------|
| `WORKER_ACCOUNT` | `worker` | RabbitMQ 帳號 |
| `WORKER_PASSWORD` | `worker` | RabbitMQ 密碼 |
| `RABBITMQ_HOST` | `127.0.0.1` | RabbitMQ 主機（Docker 裡改成 `rabbitmq`） |
| `MYSQL_HOST` | `127.0.0.1` | MySQL 主機（Docker 裡改成 `mysql`） |

### worker.py — Celery App 定義

```bash
cat crawler/worker.py
```

三個重點：
1. **`include`**：告訴 Celery 去哪找 task（`crawler.tasks`, `crawler.tasks_crawler_finmind`）
2. **`broker`**：連到 RabbitMQ 的 URL（`pyamqp://worker:worker@localhost:5672/`）
3. **logger.info()**：啟動時印出設定，方便除錯

### tasks.py — 最簡單的 Task

```bash
cat crawler/tasks.py
```

```python
@app.task()
def crawler(x):
    print("crawler")
    print("upload db")
    return x
```

只有 3 行邏輯。`@app.task()` 讓普通函式變成「可排隊的任務」。

### tasks_crawler_finmind.py — 真實爬蟲 Task

```bash
cat crawler/tasks_crawler_finmind.py
```

兩個 task：
- **`crawler_finmind_print`**：只抓資料 + print（今天用這個）
- **`crawler_finmind`**：抓資料 + 寫入 MySQL（EP08 才用）

### producer.py — 發任務

```bash
cat crawler/producer.py
```

```python
from crawler.tasks import crawler
crawler.delay(x=0)
```

只有 1 行。`crawler.delay(x=0)` = 把任務丟進 RabbitMQ，等 Worker 去做。

### producer_crawler_finmind_print.py — 批次發爬蟲任務

```bash
cat crawler/producer_crawler_finmind_print.py
```

```python
from crawler.tasks_crawler_finmind import crawler_finmind_print

for stock_id in ["2330", "0050", "2317", "0056", "00713"]:
    crawler_finmind_print.delay(stock_id=stock_id)
```

for 迴圈一次發 5 支股票的爬蟲任務。Worker 會從 RabbitMQ 取出來平行處理。
