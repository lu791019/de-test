## 第一部分：啟動 MySQL + phpMyAdmin

### Step 1：建立共用網路 my_network（若已建可跳過）

`mysql.yml`、`rabbitmq-network.yml`、Worker **全部都掛在 `my_network` 這張外部網路**，
彼此才能用「服務名稱」互相找到（例如 Worker 用 `mysql` 找到資料庫）。

```bash
docker network create my_network
# 出現 already exists 代表 EP08 已建過，忽略即可
```

### Step 2：啟動 MySQL + phpMyAdmin

```bash
cd ~/stock-crawler
docker compose -f mysql.yml up -d
```

✅ **預期**：拉取 `mysql:8.0` 與 `phpmyadmin:5.2` 映像後，兩個容器都啟動。

```bash
docker compose -f mysql.yml ps
```

看到 `mysql` 和 `phpmyadmin` 兩個服務都是 `Up`。

### Step 3：看 MySQL log 確認初始化完成

MySQL 第一次啟動要初始化資料目錄，需等十幾秒：

```bash
docker compose -f mysql.yml logs mysql | tail -20
```

✅ **預期**：看到 `ready for connections` 才代表資料庫可以連了。

> ⚠️ 常見坑：剛 `up` 馬上連線會失敗（`Connection refused`），那是 MySQL 還在初始化。
> 看到 `ready for connections` 再進行下一步。

### mysql.yml 的關鍵設定

| 設定 | 值 | 說明 |
|------|-----|------|
| `MYSQL_DATABASE` | `mydb` | 啟動時自動建立的資料庫 |
| `MYSQL_USER` / `MYSQL_PASSWORD` | `user` / `1234` | 一般使用者 |
| `MYSQL_ROOT_PASSWORD` | `1234` | root 密碼 |
| ports | `3306:3306` | 對外開放 MySQL 連線埠 |
| phpMyAdmin ports | `8080:80` | Web 介面在 `localhost:8080` |
| `PMA_HOST` | `mysql` | phpMyAdmin 用服務名稱連 MySQL |
| volumes | `mysql:/var/lib/mysql` | 資料持久化，container 重啟資料不消失 |

## 第二部分：phpMyAdmin 建資料庫

phpMyAdmin 是 MySQL 的圖形化管理介面，不用記 SQL 指令也能看資料、建庫、建表、下查詢。

### Step 1：開啟並登入 phpMyAdmin

瀏覽器打開：`http://localhost:8080`

| 欄位 | 值 |
|------|-----|
| 使用者名稱 | `root` |
| 密碼 | `1234` |

✅ **預期**：登入後左側看到既有資料庫，其中 **`mydb`** 就是 mysql.yml 自動建好的。

> 用 `root` 登入權限最完整（建庫、建表都可以）。mysql.yml 裡 root 和 user 的密碼設成一樣，方便教學。

### Step 2：認識介面

- **左側**：資料庫 / 資料表清單
- **上方頁籤**：SQL（下指令）、結構、查詢、匯出 等
- 點 `mydb` → 目前還沒有任何資料表（爬蟲跑完才會出現）

### Step 3：示範建立一個新資料庫（理解 database 概念）

雖然 `mydb` 已存在，這裡示範語法。點上方 **SQL** 頁籤，貼上並執行：

```sql
-- IF NOT EXISTS：已存在就不重建，不報錯
CREATE DATABASE IF NOT EXISTS mydb
  DEFAULT CHARACTER SET utf8mb4;   -- utf8mb4 支援中文與 emoji
```

✅ **預期**：執行成功（`mydb` 已存在，所以實際沒新增，但語法練到了）。

> 💡 一個 MySQL server 裡可以有很多 database；一個 database 裡可以有很多 table。
> 階層：**Server → Database → Table → Row / Column**。

## 第三部分：建立資料表

> 💡 重要觀念：本專案爬蟲用 `pandas.to_sql()` 寫入時會**自動建表**，
> 所以實務上你「不一定要手動建表」。但這一部分先帶你手動建一次，理解資料表結構。

### 方法 A：用 phpMyAdmin 下 SQL（推薦先練這個）

在 phpMyAdmin 上方點 **SQL** 頁籤，貼上以下指令並執行：

```sql
USE mydb;

CREATE TABLE IF NOT EXISTS TaiwanStockPrice (
    date              DATE,
    stock_id          VARCHAR(10),
    Trading_Volume    BIGINT,
    Trading_money     BIGINT,
    open              FLOAT,
    max               FLOAT,
    min               FLOAT,
    close             FLOAT,
    spread            FLOAT,
    Trading_turnover  BIGINT,
    PRIMARY KEY (date, stock_id)   -- 同一天同一檔股票不重複
);
```

✅ **預期**：執行成功，左側 `mydb` 底下出現 `TaiwanStockPrice` 資料表。

### 欄位對照（來自 FinMind TaiwanStockPrice）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `date` | DATE | 交易日 |
| `stock_id` | VARCHAR(10) | 股票代碼 |
| `Trading_Volume` | BIGINT | 成交量（股） |
| `Trading_money` | BIGINT | 成交金額 |
| `open` / `max` / `min` / `close` | FLOAT | 開 / 高 / 低 / 收 |
| `spread` | FLOAT | 漲跌 |
| `Trading_turnover` | BIGINT | 成交筆數 |

### Primary Key 的意義

`PRIMARY KEY (date, stock_id)` 是「組合主鍵」：用「日期 + 股票代碼」當每一列的唯一身分證。
好處：同一天同一檔股票不會被寫兩次、查詢更快、更新/刪除可精準定位。

> 📌 如果你想「完全交給程式自動建表」，可以**跳過手動建表**，直接進第五部分跑爬蟲，
> `to_sql()` 會依 DataFrame 欄位自動建出 `TaiwanStockPrice`（但不會有 primary key）。

### 認識常見型別

| 型別 | 存什麼 | 範例 |
|------|--------|------|
| `INT` / `BIGINT` | 整數 | `Trading_Volume = 29401639` |
| `FLOAT` / `DECIMAL(10,2)` | 小數 / 精確金額 | `close = 593.0` |
| `VARCHAR(n)` | 有長度上限的字串 | `stock_id = '2330'` |
| `DATE` / `DATETIME` | 日期 / 日期時間 | `date = '2024-01-02'` |

## 第四部分：用 Python 連線 MySQL

爬蟲程式是用 **SQLAlchemy** 連 MySQL。先用一小段程式確認本地能連得上。

### Step 1：看連線設定來自哪裡

```bash
cat crawler/config.py
```

重點：MySQL 連線參數全部用 `os.environ.get(key, default)` 讀取。

| 變數 | 預設值 | 說明 |
|------|--------|------|
| `MYSQL_HOST` | `127.0.0.1` | 本地直連用 127.0.0.1；Docker 內改成 `mysql` |
| `MYSQL_PORT` | `3306` | 連線埠 |
| `MYSQL_ACCOUNT` | `root` | 帳號 |
| `MYSQL_PASSWORD` | `1234` | 密碼 |

### Step 2：測試連線

```bash
cd ~/stock-crawler
uv run python -c "
from sqlalchemy import create_engine, text
from crawler.config import MYSQL_ACCOUNT, MYSQL_PASSWORD, MYSQL_HOST, MYSQL_PORT

address = f'mysql+pymysql://{MYSQL_ACCOUNT}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/mydb'
engine = create_engine(address)

with engine.connect() as conn:
    result = conn.execute(text('SELECT VERSION()'))
    print('MySQL 版本:', result.scalar())
    print('連線成功！')
"
```

✅ **預期**：

```
MySQL 版本: 8.0.xx
連線成功！
```

> ⚠️ 連線字串格式：`mysql+pymysql://帳號:密碼@主機:埠/資料庫`
> `pymysql` 是 Python 連 MySQL 的驅動，已透過 `uv sync` 裝好。

### Step 3：用 Python 練習 CRUD（不靠爬蟲）

先用一段最小的 DataFrame 練 `to_sql` 寫入與 `read_sql` 查詢：

```bash
uv run python -c "
import pandas as pd
from sqlalchemy import create_engine
from crawler.config import MYSQL_ACCOUNT, MYSQL_PASSWORD, MYSQL_HOST, MYSQL_PORT

engine = create_engine(f'mysql+pymysql://{MYSQL_ACCOUNT}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/mydb')

# 建一個小 DataFrame 寫進去（if_exists=replace：表存在就重建）
df = pd.DataFrame({'column_1': list(range(10))})
df.to_sql('test_upload', con=engine, if_exists='replace', index=False)
print('寫入完成')

# 讀回來
back = pd.read_sql('SELECT * FROM test_upload', con=engine)
print(back)
"
```

✅ **預期**：印出 `寫入完成` 與 0~9 的資料表。回 phpMyAdmin 重新整理可看到 `test_upload` 表。

### if_exists 三種模式

| 模式 | 表已存在時 | 適用情境 |
|------|-----------|---------|
| `fail` | 報錯，不寫入 | 怕覆蓋，要先確認 |
| `replace` | 整張表刪掉重建 | 每次都要全新資料 |
| `append` | 把新資料附加在後面 | 爬蟲持續累積（最常用） |

## 第五部分：爬蟲資料寫入 MySQL

這是本集核心。完整流程：**啟動 infra → 啟動 Worker → 發 Producer → 看 Flower → 查 DB**。

### 寫入是怎麼發生的？

看 `crawler/tasks_crawler_finmind.py` 的 `crawler_finmind` task：

```python
@app.task()
def crawler_finmind(stock_id):
    resp = requests.get(url, params=parameter)   # 1. 爬 FinMind
    data = resp.json()
    if resp.status_code == 200:
        df = pd.DataFrame(data["data"])          # 2. 轉成 DataFrame
        upload_data_to_mysql(df)                 # 3. 寫入 MySQL
```

而 `upload_data_to_mysql()` 就是用 SQLAlchemy + `df.to_sql(..., if_exists="append")`，
把整個 DataFrame 一次寫進 `TaiwanStockPrice` 表：

```python
def upload_data_to_mysql(df):
    address = f"mysql+pymysql://{MYSQL_ACCOUNT}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/mydb"
    engine = create_engine(address)
    df.to_sql("TaiwanStockPrice", con=engine, if_exists="append", index=False)
```

### Step 1：啟動基礎服務（RabbitMQ + MySQL）

MySQL 第一部分已啟動，這裡再補上 RabbitMQ：

```bash
cd ~/stock-crawler
docker compose -f rabbitmq-network.yml up -d
docker compose -f mysql.yml up -d        # 若已啟動會顯示 up-to-date
```

✅ **驗證 Web 介面**：

```bash
curl -s -o /dev/null -w "RabbitMQ  %{http_code}\n" http://localhost:15672   # 200
curl -s -o /dev/null -w "Flower    %{http_code}\n" http://localhost:5555    # 200
curl -s -o /dev/null -w "phpMyAdmin %{http_code}\n" http://localhost:8080   # 200
```

### Step 2：啟動 Worker（會連 MySQL）

```bash
docker compose -f docker-compose-worker-network.yml up -d
docker compose -f docker-compose-worker-network.yml logs | tail -30
```

✅ **預期**：兩個 Worker（`crawler_twse`、`crawler_tpex`）`ready`，並印出 `MYSQL_HOST: mysql`（不是 127.0.0.1）。
這個 compose 在 environment 設了 `MYSQL_HOST=mysql`、`RABBITMQ_HOST=rabbitmq`，讓容器用**服務名稱**連 infra。

### Step 3：發任務（會寫 DB 的 Producer）

```bash
docker compose -f docker-compose-producer-network.yml up
```

✅ **預期**：Producer 印出 `send task_2330 task` / `send task_00679b task` 後結束（exit 0）。
這支跑的是 `producer_multi_queue.py`，發的是寫 DB 版的 `crawler_finmind`。

> 一次性服務跑完 `Exited(0)` 是正常的。

### Step 4：確認 Worker 真的執行成功

```bash
docker compose -f docker-compose-worker-network.yml logs | tail -40
```

✅ **預期**：看到 `Task crawler.tasks_crawler_finmind.crawler_finmind[...] succeeded`，
而不是只有 `received` 或 `FAILURE`。

### Step 5：Flower 確認任務狀態

瀏覽器打開 `http://localhost:5555` → **Tasks** 頁籤。

✅ **預期**：看到 2 筆 `crawler_finmind`，狀態都是 **SUCCESS**。

## 第六部分：用 SQL 查詢驗證

任務成功後，資料應該已經進 `TaiwanStockPrice` 表了。**這一步是端到端驗證的關鍵**——
不能只看 Worker log 說 succeeded，要進 DB 確認真的有資料。

### 方法 A：phpMyAdmin 查詢

打開 `http://localhost:8080` → 左側點 `mydb` → `TaiwanStockPrice` → **SQL** 頁籤：

```sql
-- 1. 看資料表有幾筆（驗證資料真的進來了）
SELECT COUNT(*) FROM TaiwanStockPrice;

-- 2. 看前 10 筆
SELECT * FROM TaiwanStockPrice LIMIT 10;

-- 3. 看台積電最近 10 筆（WHERE 過濾 + ORDER BY 排序 + LIMIT 限筆數）
SELECT date, stock_id, open, close, Trading_Volume
FROM TaiwanStockPrice
WHERE stock_id = '2330'
ORDER BY date DESC
LIMIT 10;

-- 4. 各股票各幾筆（GROUP BY 聚合）
SELECT stock_id, COUNT(*) AS cnt
FROM TaiwanStockPrice
GROUP BY stock_id;
```

✅ **預期**：COUNT 大於 0；看到 2330 / 00679B 的股價資料。

### 方法 B：Python 查詢

```bash
cd ~/stock-crawler
uv run python -c "
import pandas as pd
from sqlalchemy import create_engine
from crawler.config import MYSQL_ACCOUNT, MYSQL_PASSWORD, MYSQL_HOST, MYSQL_PORT

engine = create_engine(f'mysql+pymysql://{MYSQL_ACCOUNT}:{MYSQL_PASSWORD}@{MYSQL_HOST}:{MYSQL_PORT}/mydb')
df = pd.read_sql('SELECT * FROM TaiwanStockPrice', con=engine)
print(f'總筆數: {len(df)}')
print(df.groupby('stock_id').size())          # 各股票各幾筆
print(df.head())
"
```

✅ **預期**：印出總筆數、各 `stock_id`（2330 / 00679B）的筆數，以及前幾筆資料。

> 📌 端到端閉環走完了：**爬蟲 → 任務佇列 → Worker → MySQL → 查得到資料**。
> 這就是資料工程最基本的 ingestion pipeline。

### 收尾：關閉服務

```bash
cd ~/stock-crawler
docker compose -f docker-compose-worker-network.yml down
docker compose -f rabbitmq-network.yml down
docker compose -f mysql.yml down       # 注意：data volume 會保留，資料不會消失
```

> 💡 `down` 不會刪 volume，下次 `up` 資料還在。要連資料一起清掉才加 `down -v`（謹慎使用）。
